# CHANGELOG JUNE 2026

## June 1st, 2026 - feat(lists): add ListItems chokepoint helper (items-as-objects, slice 1 of M8 foundation)

First slice of the items-as-objects work that turns List entries from plain strings (`["a","b"]` plus a parallel `item_added_at` array) into rich objects (`{id, value, added_at, label, weight, color}`). This slice is purely additive scaffolding: a single, pure, fully-tested helper class that every mutator and the upcoming schema migration will funnel through. Nothing calls it yet, so behavior is byte-identical and there is zero production risk - it is the foundation the rest of M8 stands on.

- **`app/Support/ListItems.php`**: the one place List item objects are built and validated. `make()` is the single validating constructor - value coerced to string (empty preserved, never stripped), weight clamped to a positive number defaulting to 1, color kept only if it is a valid `#rgb`/`#rrggbb` hex (else null), label kept only if a non-empty string (else null). Builders on top of it map 1:1 onto the real call sites: `appendValue()` (chat + control-driven append), `removeAt()` (draw + pop), `freshFromValues()` (store / restore / recipe install), `reconcileByValue()` (dashboard textarea save - preserves a matched item's id/age/color/weight across a reorder, mints fresh items for new values, oldest-match-wins on duplicates), `fromLegacy()` (the schema migration's string+timestamp -> object converter), and `values()` (the back-compat anchor every scalar reader leans on). The per-item `added_at` now lives inside the item, which is what lets the fragile parallel `item_added_at` array and its `ListItemTimestamps` sync discipline go away in a later slice.
- **`tests/Unit/ListItemsTest.php`**: 23 tests / 57 assertions covering the full surface - validation edges (weight clamping, color validation + lowercasing, empty-value preservation, label absence), id allocation and `next_id` reporting, the reorder/duplicate/removal semantics of `reconcileByValue`, and the legacy-conversion fallbacks. All green; Pint clean.
